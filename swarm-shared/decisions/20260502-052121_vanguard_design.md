* The absence of a README file hinders the onboarding process for new team members and makes it challenging for them to contribute to the project.
* The project's complexity and the numerous patterns and fixes applied to it are not well-documented, making it difficult for developers to understand the project's architecture and design decisions.

### Proposed change
Create a comprehensive README file that provides an overview of the project, its purpose, context, and functionality. The README file should be located in the root directory of the project (`/opt/axentx/vanguard/README.md`).

### Implementation
1. Create a new file `README.md` in the root directory of the project.
2. Add a brief introduction to the project, including its purpose and context.
3. Document the project's architecture and design decisions, including the patterns and fixes applied to it.
4. Provide information on how to navigate the project's directory structure and find specific files or components.
5. Include instructions on how to contribute to the project, including how to run tests, build the project, and submit pull requests.

```markdown
# Vanguard Project
## Introduction
The Vanguard project is a complex system that applies various patterns and fixes to improve its functionality. This project is actively being developed, and this README file provides an overview of its purpose, context, and functionality.

## Architecture
The project's architecture is based on the following components:
* Business research with knowledge-rag pipeline
* Top-hub doc insight
* Opus pr reviewer script
* Active-learning wrapper

## Patterns and Fixes
The project applies the following patterns and fixes:
* HF API rate limit bypass
* Lightning H200 not in default cloud account
* Existing Lightning Studio reuse
* Dataset-mirror writes mixed-schema files to enriched/

## Directory Structure
The project's directory structure is organized as follows:
* `/opt/axentx/vanguard/`: root directory
* `/opt/axentx/vanguard/src/`: source code directory
* `/opt/axentx/vanguard/tests/`: test directory

## Contributing
To contribute to the project, follow these steps:
1. Clone the repository
2. Run tests using `python -m unittest discover`
3. Build the project using `python setup.py build`
4. Submit pull requests to the main branch
```
