### Highest-Value Incremental Improvement
Based on the provided patterns and lessons learned, the highest-value incremental improvement that can ship in <2h is to **implement a check for Lightning Studio status before each `.run()` call and restart with `target.start(machine=Machine.L40S)` if stopped**. This fix addresses the pattern "Lightning idle stop kills training" and can be implemented quickly.

### Implementation Plan
1. **Identify the relevant code**: Locate the code that handles Lightning Studio training and identifies the `.run()` calls.
2. **Add a status check**: Before each `.run()` call, add a check to verify the status of the Lightning Studio. This can be done using the `Teamspace.studios` API.
3. **Restart if stopped**: If the studio is stopped, restart it using `target.start(machine=Machine.L40S)` before proceeding with the `.run()` call.
4. **Test the implementation**: Verify that the implementation works as expected by testing it with a stopped Lightning Studio.

### Code Snippet
```python
import lightning

# Assuming 'studio' is the Lightning Studio object
if studio.status == 'Stopped':
    # Restart the studio
    target.start(machine=Machine.L40S)
    # Wait for the studio to start
    while studio.status != 'Running':
        # Poll the studio status
        studio = Teamspace.studios.get(studio.id)
# Proceed with the .run() call
studio.run()
```
